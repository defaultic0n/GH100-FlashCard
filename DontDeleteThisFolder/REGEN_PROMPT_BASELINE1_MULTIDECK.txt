Create a complete static GitHub Pages PWA flashcard app: Baseline No.1 (Multi-Deck).

INPUT
1) File: GH-100 GitHub Admin Cert Exam QnA.docx
   - Correct answers are the options highlighted in yellow.
2) Hard deck: the 25 hard questions I previously provided.
   - Convert answer letters (A/B/C/D) into:
     - options[] objects: { text, isCorrect }
     - answers[] containing the full correct option text.

OUTPUT (ZIP — repo root only)
Required files (exact):
- index.html
- styles.css
- app.js
- service-worker.js
- manifest.json
- icons/icon-192.png
- icons/icon-512.png
- README.md
Deck system files:
- decks.json
- cards.json     (Main deck generated from DOCX)
- hard.json      (Hard deck, 25 questions)

NON-NEGOTIABLE RULES
- index.html MUST be HTML only (no JSON, no JS).
- app.js MUST be JavaScript only (no JSON).
- decks.json / cards.json / hard.json MUST be pure JSON data only (no JS, no comments).
- All assets must use relative paths (must work under https://<user>.github.io/<repo>/).

DECK SYSTEM
- decks.json structure:
  {
    "defaultDeckId": "base",
    "decks": [
      { "id": "base", "name": "Main Deck", "file": "cards.json", "description": "..." },
      { "id": "hard25", "name": "Hard Deck (25)", "file": "hard.json", "description": "..." }
    ]
  }
- UI must include a Deck dropdown (<select id="deckSelect">) in the toolbar.
- Switching decks must:
  - load the selected deck file
  - reset search box
  - keep toggles (shuffle/quiz/only missed) but apply them to the selected deck
  - track progress per-deck (progress should not mix across decks)

APP FEATURES (same UX as Baseline No.1)
- Dark UI, flip-card layout, bottom controls:
  Prev / Flip(or Reveal) / Submit (quiz) / Next
  Above: “I got it / I missed”
- Shuffle questions
- Shuffle answers (reshuffle on every navigation)
- Only missed (wrong > 0)
- Quiz mode:
  - Single-answer: auto-submit on tap
  - Multi-answer: require Submit button
  - show correct/incorrect feedback and correct answers
- Search box filters current deck only

PWA / PUBLISHING
- Update banner: “New version available” + Refresh
- service worker supports message {type:'SKIP_WAITING'}
- Cache handling:
  - Precache: index.html/styles.css/app.js/decks.json/cards.json/hard.json/manifest.json/icons
  - Bump CACHE_NAME for every build
